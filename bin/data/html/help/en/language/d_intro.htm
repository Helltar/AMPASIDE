The MIDletPascal Dialect \ Introduction


MIDletPascal as language is a dialect (a variation) of the Pascal language published by Prof. Wirth in 1970.

Case Sensibility

MIDletPascal is case-insensitive. So it doesn't matter if you write begin, BEGIN or BeGiN.

 Comments

Multi-line comments can be written between '{' and '}', for example:

{ this is a comment }

or between (* and *):

(* this is also a comment *)

Additionally, MIDletPascal supports C-style singe-line comments:

// this is a comment

and multi-line comments (since version 3.2) between /* and */ too:

/* this is also a comment */
Differences with the standard Pascal language

The MIDletPascal dialect differs on the following from the standard Pascal language:

  • procedures and functions cannot be nested, a routine cannot be declared inside the other routine
  • variable parameters in procedures and functions are not supported
  • the closing semicolon of a single-statement if-then construction can be ommitted optionally, no matter if the else keyword is present or not (WARNING: this is not fixed in the 3.x series due to backward compatibility but should be changed for the 4.0 version, so be sure to update your code to truly legal Pascal if-statements code)
  • since version 3.1, it supports an infinite loop implementation based on the repeat/forever keyword pair
  • since version 3.2, it supports C-style multi-line comments /* */
