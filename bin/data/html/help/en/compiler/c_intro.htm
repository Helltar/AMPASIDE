What's New in the MIDletPascal 3.x Compiler \ Introduction


Since Niksa Orlic (norlic) released the source code of MIDletPascal 2.02 in September 2009 under the GNU General Public License, after a successful negotiation of Sergey Naydenov (tronix286), an evolution of the original compiler has been developed by Artem (abcdef). He reverted back the compiler to a CLI stand-alone, continued working with VisualStudio, and named his releases as with three digits ("003", "005", "007", "009", etc).

One of his first public release in the boolean.name forum, was the "007" compiler version, with mainly a new lexer that allowed a few more things than the original. The most important new features are:

  • bigger array max size
  • smart string concatenation
  • shift operators
  • bytecode inlining

When the new 3.0 IDE was started in October 2009, the "007" version was the most featured one, and Javier Santo Domingo (j-a-s-d) started adapting it. The changes made were: convertion to GNU GCC Compiler 3.4.5 (using the Code::Blocks 8.02 IDE) and all the changes in the code that the port required (all made by coditional define __GNUC__); convertion to a DLL interface which involved mostly a complete main.c rewrite; addition of a callback system to allow quicker and richer reporting to the IDE; convertion of the compiler messages to the "standard pascal-way" most of the pascal compilers use; and several bugfixes that includes most of the memory leaks problems and the "literal char" bug in the new lexer reported in the sourceforge project forums. This DLL version was used by MP 3.0 ALPHA 1 to MP 3.0 BETA 2.
For the MP 3.0 BETA 3, the DLL 007 modified compiler was replaced by a modified version of the 009 stand-alone compiler (with preverifier included).
The changes made to the 009 stand-alone version are mostly the same that the ones made to the 007 to create the DLL version, with the difference that the callback system was replaced by command-line parsing (which replaced at the same time the file-based parameter handling that the Artem's 009 version had).

Since then Javier Santo Domingo (j-a-s-d) added:

  • the infinite loop support - since 3.1
  • exit keyword support - since 3.2
  • C-style multiline comment support - since 3.2
  • result keyword support - since 3.3
  • project relative libraries support - since 3.4
  • unsigned right shift operator - since 3.4
  • negative integer constants support - since 3.5
  • C-like double quoted strings support - since 3.5
